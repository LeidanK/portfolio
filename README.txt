### CONTENU DES PAGES ###

# index.html #
Cette page présente brièvement l'étudiant, ses compétences, telles que les langues parlées, les outils maîtrisés ainsi que leur niveau de maîtrise.
Cette page est la page principale du portfolio, et contient donc trois liens, chacun permettant de rediriger l'utilisateur vers l'une des trois autres pages, projets.html, contact.html, veille.html

# projets.html #
Cette page présente chaque projet que l'étudiant a pu réaliser (au nombre de 1 pour le moment). Chaque projet sera présent dans un encadré, permettant de bien les différencier.

# veille.html #
La page de veille permet de se renseigner sur les métiers de data anayst et de data scientist, en comparant les deux métiers, avec un tableau comparatif final.

# contact.html #
Cette page permet de contacter l'étudiant en renseignant son nom, son téléphone, son mail, et le message à transmettre. Certains champs sont obligatoires, comme le mail et le message, et un code JavaScript permet de vérifier la validité des informations saisies.

### INSTRUCTIONS POUR TESTER LE PORTFOLIO ###

1. Dézippez l’archive.
2. Ouvrez le fichier `index.html` dans un navigateur web compatible (Chrome, Firefox,
Edge, etc.).
3. Naviguez entre les pages à l’aide des liens.
4. Testez les fonctionnalités interactives :
	- Validation du formulaire sur la page Contact.
	- Navigation fluide entre les différentes sections.

### TECHNOLOGIES UTILISÉES ###

- HTML5 : Structure et contenu des pages.
- CSS3 : Mise en page et styles.
- JavaScript : Interactivité (validation des formulaires, animations simples).
